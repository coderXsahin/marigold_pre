# Automated Flower Disease Detection and Classification System

## Product Requirements Document (PRD)

------------------------------------------------------------------------

# 1. Project Overview

Full-stack marigold disease detection: React UI, Node.js API, FastAPI + MobileNetV2 CNN trained in Colab or locally.

------------------------------------------------------------------------

# 2. Dataset Specification

### Classes (7)

Healthy, Leaf spot, Bud rot, Botrytis blight, Foliage blight, Stem rot, White mold

### Raw data layout

    dataset/<class_name>/*.jpg

### Train / validation / test split (required)

Small datasets must be **stratified** so each split contains every class:

| Split | Ratio | Purpose |
|-------|-------|---------|
| **train** | 70% | Model learning + augmentation |
| **val** | 15% | Early stopping, hyperparameter tuning |
| **test** | 15% | Final metrics only (never used during training) |

Generated by `python notebooks/split_dataset.py` into:

    dataset_split/train/<class>/
    dataset_split/val/<class>/
    dataset_split/test/<class>/

With ~20 images/class: **14 train, 3 val, 3 test** per class.

Manifest: `dataset_split/split_manifest.json`

------------------------------------------------------------------------

# 3. Architecture

    React → Node.js → FastAPI (MobileNetV2) → Node.js → React

------------------------------------------------------------------------

# 4. ML Training

### Model

- **MobileNetV2** (ImageNet weights) + custom head — suited to small datasets
- **Single preprocessing**: `mobilenet_v2.preprocess_input` (training and inference)
- **No double rescaling** (no `rescale=1/255` in generator if using MobileNet preprocess)

### Training phases

1. Frozen base, up to 30 epochs, early stopping on `val_accuracy`
2. Fine-tune top 30 layers, 15 epochs, learning rate `1e-5`

### Outputs (`ml/`)

| File | Purpose |
|------|---------|
| `model.h5` | Model weights |
| `class_indices.json` | Index → class name |
| `model_config.json` | Architecture + preprocessing |
| `training_report.json` | Train/val/test accuracy |

### Commands

```bash
python notebooks/split_dataset.py
python notebooks/train_model.py
```

------------------------------------------------------------------------

# 5. Inference rules

1. Preprocess with MobileNetV2 `preprocess_input`
2. Softmax argmax → top class + confidence
3. If confidence **< 50%** → **Healthy** (uncertain)
4. If top class is **Healthy** → **Healthy**
5. Else → disease + `details` from `disease_info.json`

------------------------------------------------------------------------

# 6. Acceptance criteria

- [ ] Stratified train/val/test split exists
- [ ] Test accuracy reported in `training_report.json`
- [ ] Upload JPG/PNG only; invalid types error
- [ ] Predictions differ between healthy vs disease images
- [ ] Confidence shown in UI; history stores confidence
- [ ] Model loads once at FastAPI startup

------------------------------------------------------------------------

# 7. Limitations

- ~20 images/class is minimal; expect **60–85%** test accuracy at best until more data is added.
- Test set is small (3 images/class) — metrics are indicative, not production-grade.
- Retrain after any change to splits or architecture.

------------------------------------------------------------------------

# 8. Future enhancements

More data, ResNet/EfficientNet, cloud deploy, automated retraining.
